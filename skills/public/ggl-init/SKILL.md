# GGL 知识图谱初始化 Skill

## 触发条件

当满足以下全部条件时执行此流程：
1. 当前会话处于 GGL 学习模式（`agent_variant == "ggl"`）
2. `ggl.topic_graph` 为空（首次对话）
3. 用户消息包含学习主题

---

## 执行流程

### 阶段 0：提取学习主题

从用户的第一条消息中提取核心学习主题。

- 如果用户直接说 "我想学 Python"，主题就是 "Python 编程"
- 如果用户说 "帮我学习机器学习"，主题就是 "机器学习"
- 如果主题不明确，先向用户确认，然后再开始研究

**向用户说明**：你正在为该主题进行深度研究并构建个性化知识图谱，请稍等片刻。

---

### 阶段 1：并行深度研究（使用 subagent）

**必须**使用 `task` 工具启动 **3 个并行 subagent** 分别研究以下维度：

```
# Subagent 1：知识结构研究
任务：研究"[主题]"的完整知识体系和学习路径
搜索方向：
- "[主题] 知识体系"
- "[主题] learning roadmap"
- "[主题] 核心概念列表"
- "[主题] curriculum"
返回：核心知识点列表（10-20个）及其依赖关系

# Subagent 2：学习资源与难度研究
任务：研究"[主题]"各知识点的学习难度和先决条件
搜索方向：
- "[主题] 学习难点"
- "[主题] prerequisites"
- "[主题] beginner to advanced"
- "[主题] 入门到精通"
返回：各知识点的难度评级和推荐学习顺序

# Subagent 3：实践应用研究
任务：研究"[主题]"的实际应用场景和常见问题
搜索方向：
- "[主题] 应用场景"
- "[主题] real world applications"
- "[主题] common mistakes"
- "[主题] 常见错误"
返回：应用场景和常见误区，用于丰富知识节点描述
```

**并发启动**，等待所有 3 个 subagent 完成后进入下一阶段。

---

### 阶段 2：生成知识图谱

综合三个 subagent 的研究结果，构建知识图谱。

#### 节点设计原则

- **节点数量**：8~15 个，过多影响可读性，过少覆盖不足
- **粒度均匀**：避免一个大概念和多个细节概念混杂
- **节点状态**：全部初始化为 `"unvisited"`（用户尚未学习）
- **节点 ID**：使用英文小写下划线命名，例如 `linear_algebra`、`gradient_descent`

#### 边的设计原则

- 边方向表示**先决条件**：`[A, B]` 表示"学会 A 才能学 B"
- 避免循环依赖
- 入门节点（零依赖）通常是 1~3 个

#### 学习路径设计

`current_path` 应该是一条从零基础到核心能力的推荐学习序列，按照建议顺序排列所有节点 ID。

`active_node_id` 设置为路径中的**第一个节点**（最基础的入门知识点）。

---

### 阶段 3：调用 update_ggl_graph 工具

**必须调用** `update_ggl_graph` 工具将图谱写入 state：

```json
{
  "nodes": [
    {"id": "node_id_1", "label": "节点名称1", "state": "unvisited"},
    {"id": "node_id_2", "label": "节点名称2", "state": "unvisited"},
    ...
  ],
  "edges": [
    ["node_id_1", "node_id_2"],
    ...
  ],
  "active_node_id": "node_id_1",
  "current_path": ["node_id_1", "node_id_2", ...]
}
```

工具调用成功后，前端会实时显示知识图谱。

---

### 阶段 4：向用户呈现图谱概览

工具调用完成后，用结构清晰的 Markdown 向用户说明：

1. **图谱概览**：主题名称 + 节点数量
2. **推荐学习路径**：按顺序列出节点，说明为什么按这个顺序
3. **当前起始节点**：告知用户从哪里开始
4. **引导用户**：邀请用户开始第一个知识点的学习

**示例回复格式**：

```markdown
已为你构建「[主题]」知识图谱，共 N 个知识节点。

**推荐学习路径：**
1. **[节点1]** — 基础概念，学习时间约 X 天
2. **[节点2]** — 依赖节点1，...
...

**从「[节点1]」开始**，这是整个体系的基石。

你准备好了吗？告诉我你对「[节点1]」的了解程度，我会根据你的基础调整讲解深度。
```

---

## 注意事项

- **不要跳过 subagent 并行研究**：单次搜索的信息量不足以构建高质量图谱
- **图谱要有真实依赖关系**：不要随意连线，边必须反映真实的知识先后关系
- **节点命名用目标语言**：如果用户用中文提问，节点 label 用中文
- **节点 id 始终用英文**：便于代码处理和前端渲染
- **必须调用 update_ggl_graph**：不调用工具则图谱不会出现在前端
