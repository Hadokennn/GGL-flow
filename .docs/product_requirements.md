# GGL 产品需求文档

## 1. 产品概述

### 1.1 产品名称
**GGL** - 基于 deer-flow 扩展 Graph Guided Learning 功能 - 基于 Topic Graph 的自适应指导学习Agent.

### 1.2 核心哲学
> **对话是瞬时的，图谱是永恒的。对话只是修改、补充、遍历图谱的手段。**

### 1.3 产品定位
一个三层架构的 Super Agent 系统，通过 GraphRAG（图检索增强生成）技术实现自适应学习：
- **不是** "带图谱的聊天工具"
- **而是** "图谱驱动的自适应引导学习Agent"

### 1.4 核心能力
1. **自适应规划**：根据用户输入动态生成和调整学习目标图谱
2. **苏格拉底引导**：通过提问引导深度思考，而非直接给答案
3. **路径管理**：识别"顺着学"/"跳出提问"/"请求回顾"三种模式
4. **智能归属**：岔开话题时智能判断归属，可视化确认
5. **知识归纳**：将对话压缩为永恒的知识卡片

## 2. 三层产品架构

### 2.1 第一层：交互感知层 (Perception Layer)

#### 2.1.1 双视图界面
- **Linear Chat（左侧）**：线性对话流
  - 支持 Markdown、代码高亮
  - 流式响应显示
  - 消息与图谱节点关联
  
- **Dynamic Canvas（右侧）**：动态知识画布（扩展deer-flow, 支持 artifacts展示 和 图谱交互）
  - 力导向图可视化
  - 节点状态：🔵探索中 / 🟢已掌握 / ⚪未探索 / 🟡复习中
  - 学习路径高亮显示
  - 支持缩放、平移、拖拽

#### 2.1.2 双向同步机制
```
点击 Canvas 节点 ──► Chat Context 更新 ──► 三层记忆注入
     ▲                                              │
     └──────────── 对话更新图谱 ◄───────────────────┘
```

#### 2.1.3 节点交互
- **单击节点**：查看知识卡片（归纳总结）
- **双击节点**：设为当前活跃节点，更新对话上下文

---

### 2.2 第二层：逻辑编排层 (Orchestration Layer)

#### 2.2.1 意图识别引擎 (Intent Classifier)
识别四种用户意图：

| 意图 | 描述 | 系统行为 |
|------|------|----------|
| **Mastered** | 表示理解 | 点亮当前节点，引导到下一个节点 |
| **Continue** | 顺着学 | 继续当前主题，苏格拉底式提问 |
| **Digression** | 跳出提问 | 岔开话题，可能需要创建分支，压入跳出栈 |
| **Review** | 请求回顾 | 展示掌握图谱 vs 目标图谱对比 |

#### 2.2.2 学习路径管理器 (Path Manager)
```
┌─────────────────────────────────────────────────────┐
│                    目标图谱 (Target Graph)           │
│  AI 动态规划 - 应该学会的知识结构                       │
└─────────────────────────────────────────────────────┘
                          │
                          ▼ 差距分析
┌─────────────────────────────────────────────────────┐
│                    掌握图谱 (Mastered Graph)         │
│  实际学习轨迹 - 已探索/已掌握的知识                   │
└─────────────────────────────────────────────────────┘
                          │
                          ▼ 生成建议
┌─────────────────────────────────────────────────────┐
│                    当前路径 (Current Path)           │
│  正在探索的分支                                       │
└─────────────────────────────────────────────────────┘
                          │
                          ▼ 发散记录
┌─────────────────────────────────────────────────────┐
│                    跳出栈 (Digression Stack)         │
│  记录发散路径，便于引导回主线                          │
└─────────────────────────────────────────────────────┘
```

#### 2.2.3 苏格拉底引导器 (Socratic Guide)
- **自主规划**：根据目标图谱生成学习路线
- **提问引导**：通过问题引导深度思考
- **盲区检测**：识别知识盲区并针对性提问
- **回主线提示**：用户打破砂锅问到底后，提示"是否回到主线继续学习 XXX？"

#### 2.2.4 进度追踪器 (Progress Tracker)
- 对比目标图谱和掌握图谱
- 推荐下一步学习节点

---

### 2.3 第三层：知识加工层 (Knowledge Layer)

#### 2.3.1 知识抽取引擎 (Extraction Engine)
- **Topic 实体识别**：从对话中提取知识点
- **关系分析**：分析实体间的逻辑关系
- **属性抽取**：提取实体的关键属性

#### 2.3.2 归纳压缩引擎 (Summarization Engine)
将冗长对话压缩为知识卡片：
```typescript
interface KnowledgeCard {
  summary: string;           // 一句话总结
  keyPoints: string[];       // 要点列表
  examples: string[];        // 典型例子
  commonMistakes: string[];  // 常见误区
  relatedConcepts: string[]; // 关联概念
}
```

#### 2.3.3 Graph DB (Neo4j)
存储：
- 逻辑结构（实体和关系）
- 因果关系
- 学习路径
- 掌握状态

## 3. 核心交互流程

### 3.1 场景 1：初始化 (Initial)
```
用户新建学习主题
    │
    ▼
Deep Research 主题，生成调研分析报告、知识图谱、生成判断用户掌握程度的调研问卷
    │
    ▼
interrupt 用户填写调研问卷（或者回答问题）
    │
    ▼
将用户已掌握的节点更新为 Mastered、Blurry、Unknown，更新目标知识图谱
    │
    ▼
同步知识图谱到用户端canvas
    │
    ▼
引导学习图谱中的知识点
    │
    ▼
用户回答 ──► 循环
```

### 3.2 场景 2：顺着学 (Continue)
```
用户输入问题
    │
    ▼
意图识别：Continue
    │
    ▼
苏格拉底提问："那么，X 和 Y 之间有什么关系？"
    │
    ▼
用户回答
    │
    ▼
┌────────────────────────────────────────────────────────────────────┐
│  是否掌握判断：                                                      │
│  • 已掌握知识点，更新节点状态为Mastered，知识总结抽取，引导下一个知识点      │
│  • 还有疑问                                                         │
└────────────────────────────────────────────────────────────────────┘
    │
    ▼
   循环
```

### 3.3 场景 3：跳出提问 (Digression)
```
用户在学习 A 时提问 B
    │
    ▼
意图识别：Digression
    │
    ▼
┌─────────────────────────────────────────┐
│  智能归属判断：                         │
│  • 已有相关节点？→ 建议关联             │
│  • 全新话题？→ 简单回答，提示回归主题      │
└─────────────────────────────────────────┘
    │
    ▼
可视化确认弹窗
    │
    ▼
用户确认后：创建节点 + 建立关系 + 压入跳出栈
    │
    ▼
回答后提示："已理解 B，是否回到 A 的主线？"
```

### 3.4 场景 4：请求回顾 (Review)
```
用户问"我之前学过什么？"
    │
    ▼
意图识别：Review
    │
    ▼
展示：目标图谱 vs 掌握图谱对比视图
    │
    ▼
推荐未掌握的关键节点
```

### 3.5 场景 5：已掌握 (Mastered)
```
用户问"我掌握了，下一个话题吧"
    │
    ▼
意图识别：Mastered
    │
    ▼
更新节点状态为Mastered，知识总结抽取
    │
    ▼
推荐未掌握的关键节点
```
